# Internationalization (i18n) - Usage Guide

## Overview
The app now supports **English** and **French** translations using react-i18next.

## Quick Start

### Using translations in components:

```typescript
import { useTranslation } from "react-i18next";

function MyComponent() {
  const { t } = useTranslation();
  
  return (
    <div>
      <h1>{t("nav.home")}</h1>
      <p>{t("auth.login.title")}</p>
    </div>
  );
}
```

### Language Switcher
A language switcher button is available in the header. Click to toggle between FR/EN.

## Translation Files

Located in `src/i18n/locales/`:
- `fr.json` - French translations (default)
- `en.json` - English translations

## Adding New Translations

1. Add the key to both `fr.json` and `en.json`:

```json
// fr.json
{
  "myFeature": {
    "title": "Mon Titre"
  }
}

// en.json
{
  "myFeature": {
    "title": "My Title"
  }
}
```

2. Use in component:

```typescript
const { t } = useTranslation();
<h1>{t("myFeature.title")}</h1>
```

## Language Detection

The app automatically:
1. Checks localStorage for saved language preference
2. Falls back to browser language
3. Defaults to French if no match

## Changing Language Programmatically

```typescript
import { useTranslation } from "react-i18next";

const { i18n } = useTranslation();
i18n.changeLanguage("en"); // Switch to English
i18n.changeLanguage("fr"); // Switch to French
```

## Best Practices

1. **Use nested keys** for organization:
   ```json
   {
     "auth": {
       "login": {
         "title": "Login"
       }
     }
   }
   ```

2. **Keep translations consistent** across both languages

3. **Use descriptive keys**: `auth.login.submitButton` not `button1`

4. **Test both languages** when adding new features

## Database Integration (Future)

You can store user language preference in the database:

```typescript
// User table
{
  "id": 1,
  "username": "johndoe",
  "preferredLanguage": "fr" // or "en"

```

Then apply on login:

